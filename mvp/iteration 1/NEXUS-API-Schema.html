JWT → X-Tenant-ID Extraction Flow
1
Client obtains JWT from Okta containing claims: tenant_id, sub, roles, exp
2
Request arrives at Kong proxy :8000 with Authorization: Bearer <JWT>
3
Kong JWT plugin validates signature via Okta JWKS, verifies exp claim — rejects with 401 if invalid
4
Request-transformer plugin extracts jwt_claims.tenant_id → injects as trusted header X-Tenant-ID
5
Rate-limiting plugin applies 100 req/min quota scoped to X-Tenant-ID (not IP) — tenant-isolated throttling
6
Upstream microservice receives request with X-Tenant-ID, X-Correlation-ID, X-User-ID — never decodes JWT

Service-to-Service Interaction Map
M6 UI → Kong :8000 JWT → M2 Query API :8003
POST /api/v1/query · Body: {query, session_id}
Headers injected by Kong: X-Tenant-ID, X-Correlation-ID, X-User-ID
Response: {session_id, status: "processing"} — result arrives via WebSocket
M2 Safety Agent → Direct HTTP → OPA :8181
POST /v1/data/nexus/query/allow
Input: {tenant_id, user_id, query_intent, entities_accessed}
Output: {allow: true|false, reason} — blocks execution if false
M2 Agents → Direct HTTP (internal) → M3 AI DB
POST /internal/v1/vector/search — Pinecone semantic search
POST /internal/v1/graph/query — Neo4j Cypher execution
GET /internal/v1/timeseries/{entity_id} — TimescaleDB fetch
All calls include tenant_id as query param for RLS enforcement
M1 CDM Mapper → Kafka async → M4 MEQ
Topic: {tid}.m1.mapping_failed
Payload: {source_system, source_table, source_field, confidence, proposed_cdm_field}
M4 MEQ exposes via GET /api/v1/governance for human review in M6
M4 Governance API → Kafka async → M1 CDM Mapper
Topic: {tid}.m4.mapping_approved
Triggers CDM Mapper cache invalidation within <10 seconds
Approved mapping promoted from Tier-2 to Tier-1 (silent processing)
M1 AI Store Router → Kafka async → M3 Writers
Topic: {tid}.m1.ai_routing_decided
Payload includes: entity_type, target_stores[], cdm_entity
Routing: party/employee → vector+graph · transaction → graph+timeseries · product → vector
M2 Exec Agent → Kafka async → M6 WS Relay
Topic: {tid}.m2.agent_response_ready
Payload: {session_id, response_text, sources[], reasoning_trace[], safety_approved}
M6 WS relay forwards to correct WebSocket connection by matching session_id
M1 Drift Detector → Kafka async → M2 Structural Agent
Topic: {tid}.m1.schema_drift_detected
Payload: SourceKnowledgeArtifact {schema_snapshot_id, tables, captured_at}
Triggers M2 Structural Agent to re-interpret semantic schema changes
